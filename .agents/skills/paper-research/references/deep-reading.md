# 论文中文对照与递进精读方法

本文件固定“怎么读”，不固定学科内容。每篇论文都产出两份用途不同的文稿，再分别导出 PDF。两份文稿共用同一次 PDF 提取、章节边界、图表裁剪和来源哈希，禁止为了生成第二份文稿重复解析全文。

## 1. 两份产物

### 中文全文对照版

目标是让不熟悉英文的读者能够按原论文顺序读懂全文，并能随时回到原文核查。

- 文件：`translations/<citation_key>.md`，导出为 `<短标题>_中文全文对照.pdf`。
- 从标题、摘要、引言开始，覆盖正文、结论和与结论有关的附录；参考文献条目保留原文，不逐条翻译。
- 按原论文的章节与页码对齐。每个翻译块必须标注 `原文 PDF 第 N 页｜原文章节 X`。
- 每个翻译单元依次放置 `英文原文` 与 `中文逐句完整翻译`，使用完全相同的单元 ID 一一对应。不得合并、概括、删减或跳过原文句子；段落过长时可以拆成多个连续单元，但不能改变顺序。
- AUTO 区必须包含 `PAPERLAB:TRANSLATION` 覆盖账本。`source_units` 与 `translated_units` 必须等于实际成对单元数，`omitted_units` 必须为 `0`；否则禁止导出 PDF。
- 保留原章节编号、图号、表号、公式号、脚注号和文内引用。图表标题翻译为中文，英文术语首次出现时放在括号中。
- 翻译是忠实转换，不是总结、改写或评论。原文重复、保守或存在跳跃时，也不得擅自补成更强结论。
- 无法可靠提取、公式缺字、双栏错序或语义不确定时，明确写 `无法可靠确认`，并给出原文 PDF 页码。
- 首页注明：`机器辅助翻译，原文为准`。
- 只插入理解正文所需的原图、表和公式，不把每个原文页面截图塞进文稿。这样保留对照能力，也避免重复图片和 token 浪费。

### 精读学习版

目标是带着读者沿作者的论证顺序学完论文，而不是把论文拆成互不相连的分类摘要。

- 文件：`papers/<citation_key>.md`，导出为 `<短标题>_精读学习.pdf`。
- 正文目录跟随原论文目录；可以加中文教学副标题，但不得改变论证先后。
- 不得用“问题、方法、结果、贡献”重新横向切块。论文若先提出假设、再设计实验、再回到机制解释，学习稿也按这个次序推进。
- 重要数字和判断直接在解释后标注 `（原文 PDF 第 N 页，图/表/式 X）`，不要把来源全部挪到一个孤立表格里。
- 参数、数据集字段、实验步骤和复现条件可以用表格或短列表；主论证必须用完整段落和过渡句。
- 不设置固定页数或字符上限。长度由论文内容和读者理解所需决定。

## 2. 固定精读路线

### 第一步：阅读准备

用简短正文说明：论文出现前领域处于什么状态，读者需要先懂哪些概念，本次阅读要解决什么问题。随后给出按章节排列的学习路线，而不是先给最终结论清单。

### 第二步：画出作者论证路线

根据原文目录建立 5-10 个连续节点。每个节点应能接成一句因果链：已有方法留下什么问题，作者因此提出什么，如何验证，证据又把论证推进到哪里。

### 第三步：沿原文章节逐节学习

每个章节用原文标题和页码作为主锚点。章节内部按自然段或小节推进，并持续回答：

1. `上一步留下的问题` 是什么。
2. 作者在这里新增了哪个概念、假设、结构或实验。
3. 图、表、公式或数据具体怎样支持这一步。
4. `这一节回答了什么`，证据强度到哪里为止。
5. `下一节为什么出现`，它要解决当前仍未解决的什么问题。

这些提示应融入自然叙述，不要机械地把五句话做成五个重复栏目。

### 第四步：图表和公式就地教学

图表必须紧跟首次解释它的段落。先说明读图目的，再解释坐标、变量、单位、图例、比较条件和主要趋势，最后说明它支持或不能支持什么。公式先说明它为什么在此处出现，再解释符号、假设、单位和它对下一步推理的作用。

### 第五步：把实验写成证据链

实验不能只报数字。先说作者需要排除哪种替代解释，再说对照如何设置、改变了什么、观察到什么，最后说明这项结果把论证推进了多少。最佳结果如果包含预训练、集成、多裁剪或特殊数据处理，必须和数字一起写出。

### 第六步：重建完整论证链

读完原文最后一个正文部分后，用连贯文字重新走一遍：研究情境如何导出问题，问题如何导出设计，设计如何产生证据，证据如何支持结论。明确区分作者明确陈述、数据直接支持、合理推断、Codex 的解释和无法确认。

### 第七步：形成学习闭环

最后回答读者现在真正掌握了什么、最容易误解什么、哪些边界必须记住、回看哪几张图能恢复整篇论文，以及下一篇论文为什么应该接在这里。来源核查索引放在附录，不打断正文学习。

## 3. 并行精读协议

批量精读使用“协调代理 + 每篇论文一个子代理”。固定方法相同，论文内容和章节标题不同。

### 协调代理的职责

1. 锁定用户指定的论文集合，不擅自减少篇数。
2. 为每篇论文确定 citation key、源 PDF、SHA-256、项目问题和唯一输出目录。
3. 根据平台可用槽位并发启动；槽位不足时分波次，例如 6 篇、4 个槽位按 `4 + 2` 执行。
4. 验收每篇译文、学习稿、证据卡和图片，确认页码覆盖与原 PDF 哈希。
5. 串行更新项目共享状态并统一备份。

### 子代理的职责与写入边界

每个子代理只处理一篇论文，只能写入：

```text
translations/<citation_key>.md
papers/<citation_key>.md
evidence/<citation_key>.json
assets/<citation_key>/
```

子代理不得修改 `PROJECT.md`、`STATUS.md`、`state/<project_id>.json`、事件日志、`synthesis.md` 或其他论文文件，不得执行 Git、checkpoint 或 Zotero 写操作。完成后返回简短清单：源哈希、读取页段、生成文件、未确认内容和错误。

### 失败隔离

- 一个论文失败不得取消其他论文。
- 已完成论文立即保留；失败论文进入待重试，不要求成功论文重跑。
- 子代理写入自己的唯一文件，不共享临时 Markdown 或图片目录。
- 协调代理在验收成功后才更新 processed citation keys；一轮结束后做一次私有备份。

## 4. 验收清单

- Obsidian 论文节点的第一组阅读链接是原文正文及实际使用的补充材料，随后才是中文对照稿和精读稿；不得只列生成稿。
- Zotero 附件使用 `zotero://open-pdf/library/items/<attachment-key>`，尚未入库的合法本地来源使用只读 `file:///` 链接；逐项验证链接可打开，不复制源 PDF。
- 中文全文对照版从摘要到结论无静默漏页，所有翻译块可回到原文 PDF 页码；英文原文与中文逐句完整翻译按 ID 一一对应，`source_units == translated_units` 且 `omitted_units == 0`。
- 精读学习版目录与作者论证顺序一致，页码锚点总体单调向后推进。
- 论文后半部分的系统、附录或反例若参与结论，不得只讲前半部分的知名模型。
- 任意抽查三个数字，都能定位到页码、图号、表号或公式号，并保留样本、单位和条件。
- 图像可见、文字可搜索。
- 原始 Zotero PDF 处理前后哈希完全一致。
- 多论文批次中，成功、失败、待重试和下一波次均有明确计数。
