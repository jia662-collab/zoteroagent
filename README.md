# Zotero × Codex 论文研究助手

这是一套以 Zotero 为文献主库、以 Codex 为研究助手的长期论文工作流。每个 Codex 任务窗口负责一个研究课题，搜索、筛选、精读和综合结果会持续保存；窗口关闭或上下文压缩后仍可恢复。

## 你只需要这样说

```text
开始研究柔性量子点光电探测器
搜索近五年的综述和代表性论文
把第 2、5、7 篇导入 Zotero
继续
精读这三篇核心论文
比较方法并整理研究缺口
```

Codex 会自动处理项目文件、文献索引、PDF 分页、状态检查和私有备份。你不需要记忆 Python、Git 或 PaperLab 命令。

## 第一次使用

只需要完成两项一次性设置：

1. 在 Zotero 安装 Better BibTeX，并将自动导出位置设为本项目的 `bibliography/library.bib`，勾选 `Keep updated`。
2. 保持 Zotero 中的标题、作者、年份、出版物、DOI 和 PDF 附件准确。

私有研究数据保存在本项目内的 `research/`，它是独立的私有 Git 仓库；公开的工具仓库不会看到其中内容。

## 开始一个研究课题

在新的 Codex 任务中直接说：

```text
开始研究：你的研究主题或问题
```

Codex 会建立独立项目，询问至多一个真正影响检索范围的问题。以后这个窗口只负责该课题，不会与其他研究窗口混在一起。

## 搜索和导入论文

说“搜索相关论文”后，Codex 默认给出最多 15 篇经过来源核验和去重的候选论文。你只需选择编号：

```text
把第 2、5、7 篇导入 Zotero
```

Codex 会生成 `zotero_import.ris`。在 Zotero 中导入该文件后，Better BibTeX 会自动刷新文献索引。

## 筛选和精读

说“继续”即可进入下一阶段。Codex 会先核对 Zotero 导出，再进行筛选。需要精读时直接说论文编号、标题或 citation key。

PDF 按页分批读取，不会一次把整篇论文塞进上下文。每篇笔记都会保存来源位置，并区分作者陈述、数据证据、推断和无法确认的内容。

## 比较和知识综合

说“比较这些论文”“整理争议”或“寻找研究缺口”即可。Codex 优先读取每篇论文的短证据卡，只有核查具体数值或争议时才回到原文页，因此文献增加后也不会每次重读整个资料库。

## 恢复长期项目

原任务窗口中直接说：

```text
继续
```

如果需要用新窗口替换旧窗口，说：

```text
恢复项目：项目名称
```

研究目标、当前阶段、下一步和已生成结果都保存在私有研究仓库，不依赖聊天记录。

## 数据安全

- Zotero 数据库、storage 和原始 PDF 始终只读。
- 不绕过付费墙、验证码或版权限制。
- 自动分析不会覆盖“人工确认”内容。
- 原始 PDF、BibTeX、本机 Zotero 路径和全文缓存不会进入 GitHub。
- 公开仓库只保存通用工具；研究项目进入独立私有仓库。

## 结果在哪里

每个课题位于：

```text
research/projects/<项目编号>/
```

其中包含研究目标、当前状态、搜索结果、Zotero 导入文件、筛选结果、逐篇笔记、证据卡和综合报告。

## 技术维护

这一节只供维护工具时使用，日常研究不需要执行。

环境要求：Python 3.11 或更高版本、PyMuPDF、pypdf、pytest。内部执行器位于 `.paperlab/paperlab.py`，仓库级工作流位于 `.agents/skills/paper-research/`。

运行自动测试：

```powershell
python -m pytest
```
