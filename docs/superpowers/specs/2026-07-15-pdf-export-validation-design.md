# PDF 导出自动验收设计

## 目标

后续 PaperLab PDF 导出不得再次把 LaTeX 公式源码直接印到页面上。用户继续使用原有 `export` 命令，不增加手工步骤。

## 方案

所有导出继续经过 `.paperlab/paperlab.py` 的共享 `export_markdown_pdf`：

1. 先把 Markdown 与公式渲染为临时 PDF。
2. 在临时 PDF 上检查页数、空白页、Unicode 替换字符和常见 LaTeX 源码残留。
3. 检查失败时返回明确错误且不替换已有 PDF；检查通过后再原子替换目标文件。
4. 在 `paper-research` 导出约定中注明该自动验收要求。

## 验证

- 一个回归测试证明含公式的正常稿件能够导出且没有源码残留。
- 一个回归测试证明包含残留公式源码的异常 PDF 会被拒绝。
- 全量测试通过后，用当前四篇论文的 8 份 PDF 做一次真实扫描。

## 不做

- 不增加独立 QA 命令、配置文件或新依赖。
- 不要求用户逐页手工确认。
- 不修改原始 PDF、Zotero 文件或 Markdown 人工确认区。
