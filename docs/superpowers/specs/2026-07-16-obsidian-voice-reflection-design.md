# Obsidian 语音个人理解笔记设计

## 目标

学习者阅读一个知识点时，可以在右侧并排打开该知识点专属的“我的理解”笔记，使用 VoicePaste 把想法直接转换为文字。系统只保存文字，不保存录音。

设计必须满足：

- 正式知识、论文证据、实验内容与个人想法分开存放。
- 每个知识点只有一份长期积累的个人理解笔记。
- 自动更新知识库时不得覆盖个人文字。
- 不让个人笔记重新污染全局知识图谱。
- 不安装新的 Obsidian 社区插件，不引入音频处理或转写服务。

## 现有条件

- 知识点页由 PaperLab 工作流生成，并使用受控自动内容区。
- 每个知识点当前已有不会被自动覆盖的人工笔记区。
- Obsidian 已启用核心 Templates 插件。
- Obsidian 原生支持左右分栏、内部链接和 Markdown 笔记嵌入。
- VoicePaste 2.1.3 已安装，可通过全局快捷键把语音转写并粘贴到当前文本位置。

## 选定方案

采用“一知识点一份独立个人理解笔记”。知识正文位于左侧，个人理解笔记在右侧分栏打开。

未采用的方案：

- 直接把语音内容追加到知识正文：正文和个人想法容易混杂。
- 使用统一语音收件箱后自动分类：增加分类步骤和误链接风险。
- 保存录音并使用 Whisper 转写：用户只需要文字，该链路没有必要。

## 文件结构

个人理解集中放在学科目录的 `09_我的理解` 下，并保留原模块层级：

```text
10_深度学习与CNN/
├─ 02_神经网络基础/
│  └─ 01_人工神经元与网络层.md
└─ 09_我的理解/
   └─ 02_神经网络基础/
      └─ 01_人工神经元与网络层-我的理解.md
```

命名规则为：

```text
<知识点原文件名>-我的理解.md
```

同一知识点的多次语音输入追加到同一文件，不为每次学习创建新文件。

## 知识正文入口

每个知识点的自动内容区加入固定入口：

```markdown
> [!tip] 我的理解
> [[10_深度学习与CNN/09_我的理解/02_神经网络基础/01_人工神经元与网络层-我的理解|在右侧打开个人理解笔记]]
```

入口由工作流维护。个人理解文件由学习者维护，工作流只能在文件不存在时创建，不得覆盖已存在文件。

## 个人理解模板

```markdown
---
type: reflection
concept: "[[10_深度学习与CNN/02_神经网络基础/01_人工神经元与网络层]]"
status: 待提炼
tags: [reflection]
---

# 我的理解：人工神经元与网络层

## 语音记录

### 2026-07-16 20:30

使用 VoicePaste 快捷键开始表达。

## 提炼后的理解

- 一句话理解：
- 我的例子：
- 与已有知识的联系：
- 仍然不明白：
```

模板不包含“后续验证”。`status` 只使用 `待提炼` 和 `已提炼`，避免形成额外任务系统。

## 学习流程

1. 在左侧打开一个知识点。
2. 从“我的理解”入口打开对应笔记，并选择右侧分栏。
3. 在“语音记录”下插入当前日期和时间。
4. 将光标放在时间标题下，使用 VoicePaste 快捷键开始表达。
5. 停止语音输入后保留原始转写。
6. 在“提炼后的理解”中整理一句话理解、个人例子、知识联系和疑问。

原始语音转写与提炼内容共存。任何后续自动整理只能修改“提炼后的理解”，不得删除或覆盖“语音记录”。

## 知识网络规则

- 正式知识点继续使用 `#knowledge`。
- 个人理解只使用 `#reflection`，不使用 `#knowledge` 或其子标签。
- 个人理解必须链接回唯一的正式知识点；正式知识点也必须链接个人理解。
- 默认全局图谱不显示 `#reflection`，避免每个知识点增加卫星节点后重新变得拥挤。
- 查看个人想法时使用局部图谱或搜索 `tag:#reflection`。

## 工作流边界

实现只需要扩展现有课程和 Vault 生成逻辑：

1. 根据现有知识点清单生成缺失的个人理解文件。
2. 给每个知识点生成稳定的个人理解链接。
3. 创建一份供未来知识点使用的核心 Templates 模板。
4. 创建“我的理解总览”，按待提炼和已提炼列出个人笔记。
5. 在迁移和同步测试中验证个人文字不被覆盖。

不增加社区插件、自定义 Obsidian 插件、数据库、录音目录、音频附件、后台服务或自动语义分类。

## 异常处理

- VoicePaste 没有文字：先检查光标、VoicePaste 运行状态、麦克风权限、输入设备和当前语音模型。
- 转写出现术语错误：保留原始文字，在提炼区修正，不修改正式知识正文。
- 个人理解文件已存在：工作流跳过创建，只检查双向链接。
- 知识点更名或移动：迁移映射同时更新入口和 `concept` 链接，但不改个人正文。
- Obsidian Sync 产生冲突副本：保留两份，由人工合并；自动流程不得选择性删除。

## 验收标准

- 每个正式知识点都有且只有一份对应的个人理解笔记。
- 每对文件具有可解析的双向链接，没有孤立或失效链接。
- 知识正文与个人理解可以左右并排使用。
- 从知识点进入可输入状态不超过两次界面操作，之后使用 VoicePaste 说话。
- Vault 中不产生新的音频文件。
- 连续运行两次生成、迁移和同步流程，第二次为零重复改写。
- 在个人理解中写入测试文字后重新运行工作流，测试文字逐字保留。
- 默认全局知识图谱仍只突出正式知识结构。
